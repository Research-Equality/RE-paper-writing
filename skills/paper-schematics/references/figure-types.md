# Figure Types

Pick the figure type from the communication goal.

## Graphical Abstract

- Use when the paper needs a one-image summary for the full story.
- Best for: title-page figure, journal TOC image, first-pass reviewer orientation
- Include: problem, method, key outcome, and takeaway
- Avoid: dense paragraphs, tiny labels, multiple unrelated stories

## Method or System Overview

- Use when the pipeline or architecture is hard to hold in working memory from text alone.
- Best for: multi-stage methods, data flow, training and inference paths, human-in-the-loop systems
- Include: stages, inputs, outputs, and decision points

## Reporting Flow Diagram

- Use when the study needs transparent inclusion, exclusion, screening, or allocation logic.
- Best for: PRISMA, CONSORT, cohort construction, dataset filtering
- Include: counts, reasons for exclusion, and final analyzed sets

## Mechanism or Concept Figure

- Use when the contribution is an explanatory idea rather than a numeric comparison.
- Best for: causal hypotheses, biological pathways, conceptual frameworks
- Include: only the entities and interactions needed for the paper's argument

## Multi-Panel Explanatory Figure

- Use when one visual must combine overview and detail.
- Common pattern:
  - Panel A: overview
  - Panel B: component detail
  - Panel C: example or failure mode
- Keep each panel doing a distinct job.
