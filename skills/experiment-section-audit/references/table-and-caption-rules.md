# Table and Caption Rules

Use these rules when auditing experiment communication.

## Table Rules

- Put captions above tables.
- Use `booktabs`, not vertical lines or dense `\hline` stacks.
- Show metric direction in headers, for example `Acc ↑` or `WER ↓`.
- Keep precision consistent within a column.
- Let one table communicate one message.
- Use restrained highlighting for best or second-best values only.

## Caption Rules

- State the experimental setting, protocol, or notation that the reader needs.
- Keep captions short; discussion belongs in the body text.
- Make the table readable without forcing the reader to search for basic context elsewhere.

## Reviewer Checks

- Would a reviewer understand what this table proves in under 10 seconds?
- Does the caption explain the setup without becoming a mini paragraph?
- Are multiple unrelated stories being mixed into one table?
- Are key comparisons visually easy to find?
