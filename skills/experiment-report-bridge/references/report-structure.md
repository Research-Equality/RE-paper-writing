# Report Structure

Use a compact, paper-facing Markdown report rather than a lab notebook dump.

## 1. Summary and Goals

- What question did the experiment try to answer?
- What was the success signal?
- What is the one-sentence outcome?

## 2. Experiment Plan

- Stages of the run
- Key checkpoints
- Planned comparisons and stopping criteria

## 3. Setup

- Data and splits
- Model or method configuration
- Environment and critical parameters
- QC checks actually run

## 4. Baselines and Comparisons

- Which baselines were intended?
- Which ones actually ran?
- Which comparisons are still missing?

## 5. Results

- Main metrics with exact paths to tables or figures
- Confidence intervals, error bars, or correction notes when applicable
- Any failed or inconclusive runs that matter for interpretation

## 6. Analysis, Limitations, and Next Steps

- What the evidence supports
- What it does not support yet
- Clear follow-up experiments or fixes

## 7. Sources

- Include only if external web research or outside references were used during analysis
